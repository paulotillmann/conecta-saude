## Diretrizes Principais
Toda tarefa de criação, modificação ou correção de componentes, páginas e fluxos do projeto **CONECTA-Saúde** deve estar alinhada com as seguintes regras de desenvolvimento:

1. **Mobile-First**: Iniciar toda estrutura visando dispositivos móveis e expandir progressivamente para desktop.
2. **Responsividade & Fluidez**: Utilizar layouts fluidos, CSS Grid, Flexbox e unidades relativas (`rem`, `em`, `vh`, `vw`, `%`), evitando larguras e alturas fixas rígidas.
3. **Acessibilidade (WCAG)**: Contraste de cores correto, HTML semântico, foco visível, suporte a teclado e leitores de tela.
4. **Performance**: Skeleton loaders, lazy loading de imagens, otimização de renderização.
5. **Segurança & Privacidade**: Componentização segura, controle de permissões de visualização e ocultação lógica no frontend (nunca ocultar dados sensíveis apenas com CSS).

Consulte o arquivo completo de especificações em [SKILL.md](file:///C:/CONECTA-Saude/SKILL.md) antes de cada alteração de código.
